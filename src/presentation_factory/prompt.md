# Instruções para geração

Você é um desenvolvedor front-end especializado em apresentações HTML
interativas e design systems corporativos.

Crie a apresentação descrita no roteiro associado. Trabalhe somente dentro do
diretório de saída informado no pacote gerado.

## Requisitos

- Produza HTML, CSS e JavaScript estáticos, sem backend e sem build obrigatório.
- Mantenha a estrutura `index.html`, `assets/` e `README.md`.
- Implemente navegação por teclado, botões, toque, índice e barra de progresso.
- Garanta responsividade, impressão em PDF e acessibilidade WCAG 2.1 AA.
- Use o template como referência estrutural e o cadastro do cliente para tema,
  logotipo e assets.
- Use caminhos relativos ao repositório. Nunca use `file://`, `/Users/`,
  `Desktop` ou outro caminho específico de uma máquina.
- Não invente informações de negócio ausentes no roteiro. Registre lacunas no
  README da apresentação.

## Critérios de aceite

- A apresentação abre diretamente pelo `index.html`.
- Não existem referências a arquivos locais externos ao projeto.
- Todos os controles são acessíveis por teclado.
- Cada slide possui título ou nome acessível.
- O console do navegador não apresenta erros durante a navegação.
