# Checklist de qualidade do deck

## Narrativa

- [ ] Objetivo e público claros.
- [ ] Mensagem central identificável.
- [ ] Modelo principal e padrões secundários escolhidos pelo objetivo, não por
      hábito.
- [ ] Decisão ou próximo passo explícito.
- [ ] Um foco por slide.
- [ ] Títulos comunicam conclusões.
- [ ] Fatos e hipóteses diferenciados.
- [ ] Lacunas sinalizadas.

## HTML

- [ ] Idioma correto.
- [ ] Tema claro declarado.
- [ ] IDs únicos.
- [ ] Títulos associados aos slides.
- [ ] Controles semânticos.
- [ ] Botão de tela cheia presente e acessível.
- [ ] Botão de loop/autoplay presente e acessível.
- [ ] Tabelas e imagens acessíveis.
- [ ] Imagens e logos vêm de `assets/` do deck final.
- [ ] Paths relativos.

## CSS

- [ ] Base de 18px.
- [ ] Tokens consistentes.
- [ ] CSS separado cobre todos os elementos visíveis do HTML.
- [ ] Fontes, tamanhos, pesos, line-height e larguras foram ajustados slide a
      slide.
- [ ] Contraste adequado.
- [ ] Foco visível.
- [ ] Sem overflow indevido.
- [ ] Breakpoints testados.
- [ ] Reduced motion.
- [ ] Print por slide.
- [ ] Nenhum componente importante depende do estilo padrão do navegador.

## JavaScript

- [ ] Sintaxe válida.
- [ ] Botões funcionam.
- [ ] Teclado funciona.
- [ ] Atalho `F` alterna tela cheia.
- [ ] Atalho `L` alterna loop/autoplay.
- [ ] Hash funciona.
- [ ] Índice sincronizado.
- [ ] Contador e progresso corretos.
- [ ] Swipe não conflita com scroll vertical.
- [ ] Console sem erros.

## Conteúdo

- [ ] Sem dados inventados.
- [ ] Sem texto excessivo.
- [ ] Sem fonte pequena para compensar densidade.
- [ ] README registra uso, assunções e lacunas.
